## 🎬미니DB프로젝트:영화 정보 시스템 구축 
일정: 2026-01-29 ~ 2026-01-30

1. 프로젝트 개요
실제 영화 정보 사이트의 순위 20위 데이터를 기반으로 데이터베이스 모델링 및 구축을 진행했습니다.

2. 주요 수행 단계
- 설계: eXERD를 활용한 논리/물리 데이터 모델 생성

- 구현: 파트별 DDL(정의어) 및 DML(조작어) 작성

- 배포: Oracle DB 서버 실제 구현 및 테스트

3. 담당 역할 및 업무
저는 사용자 및 리뷰 관리 파트의 스키마 설계와 데이터 구축을 담당했습니다.

- 대상 테이블: User, UserReview, Expert, ExpertReview

- 수행 내용: 해당 테이블의 DDL 작성 및 초기 데이터(DML) 생성

💡 프로젝트 회고 및 배운 점
✅ 기술적 성장: 데이터 모델링의 이해
- N:N 관계의 해소: 과거 독학 시 자주 범했던 N:N 관계 설계의 문제점을 파악했습니다.

- 중간 테이블 활용: 데이터 무결성을 위해 N:N 관계를 1:N, N:1로 분리하는 중간 테이블 설계 기법을 실무에 적용해 보았습니다.

✅ 협업 경험 및 문제 해결
- 설계 수정의 어려움: 역설계 과정에서 미처 발견하지 못한 컬럼들이 뒤늦게 추가되어 구조를 수정하는 시행착오를 겪었습니다. 이를 통해 초기 설계 단계의 정밀함이 얼마나 중요한지 체감했습니다.

- 커뮤니케이션: 제한된 시간 내 완성도를 높이기 위해 팀원들과 범위를 조율(Scope 조정)하며 협업하는 방식을 익혔습니다. 작업 방식의 통일과 의견 취합이 프로젝트 속도에 미치는 영향을 깊이 이해하게 된 계기가 되었습니다.
