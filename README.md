## 미니DB프로젝트 (2026-01-29 ~ 2026-01-30)

오라클 sql 수업을 진행하며 29일 오후 ~ 30일 까지 미니DB프로젝트를 진행하였다.

과정은 영화정보가 있는 사이트의 영화순위 20위까지를 대상으로 

eXERD, Oracle, sqlDeveloper 등을 이용하여

1. 논리데이터모델 생성
2. 물리데이터모델 생성
3. 각자 맡은 구역 DDL 생성
4. 각자 맡은 구역 DML 생성
5. DB에 구현

의 과정으로 진행되었다.

내가 맡은 구역은 user, userReview, expert, ExpertReview 파트의 DDL과 DML 작성이었다.

이미 만들어져 있는 정보를 역순으로 DB에 구현하려다 보니, 초반에 보이지 않던 컬럼들이 발견되어 수정해야하는 어려움이 있었다.

제한된 시간안에 완성하는게 중요하다고 생각하였고, 모든 영화목록을 볼수는 없었기에 팀원들과 상의하에 생략하게 된 부분들도 많았다.

처음으로 협업을 해본 결과, 동료들과 작업방식을 통일시키고 설계단계에서 의견을 취합하는 것이 생각보다 어렵다는 걸 알 수 있었다.

개인적으로 배우게 된 점은, 독학으로 코딩을 했었을때 N:N관계의 테이블을 많이 만들었었다는 걸 알게되었고 

그 해결방법으로 1:N, N:1관계로 분리시키기 위해 중간테이블을 설계하는 법을 알게되었다.
